## 🐳 Шаг 1: Установка Docker Desktop
### 🪟 Инструкция по установке Docker Desktop на Windows

#### 1. Скачивание и установка программы
1. Скачайте официальный инсталлятор: [Docker Desktop для Windows](https://docker.com).
2. Запустите установщик. При установке обязательно убедитесь, что включена галочка **«Use WSL 2 instead of Hyper-V (recommended)»**.
3. Перезагрузите компьютер по требованию установщика.
4. Запустите Docker Desktop с рабочего стола и дождитесь, пока индикатор в левом нижнем углу окна программы станет зеленым (`Engine Running`).

#### 2. Подготовка и обновление WSL 2
Чтобы Docker Desktop работал стабильно и без сбоев, необходимо обновить ядро подсистемы Linux (WSL) в Windows до самой актуальной версии. Для этого откройте **PowerShell от имени администратора** и выполните команду:

```powershell
wsl.exe --update
```

После завершения обновления рекомендуется перезапустить Docker Desktop. Теперь ваша Windows полностью готова к запуску игровых Linux-контейнеров!

## 🎮 Шаг 2: Загрузка сервера

Когда Docker полностью готов к работе, откройте cmd, скачайте официальный образ приватного сервера командой:

```bash
docker pull n0uk/dynast.io-server-private:release
```
Ссылка на оффициальный docker репозиторий разработчика игры: https://hub.docker.com/r/n0uk/dynast.io-server-private

---

### Создание папки сервера
Создайте папку с файлами сервера (к примеру папку `dyn_private` на рабочем столе), скачайте файлы своего сервера (в качестве примера в репозитории размещенны стандартные файлы сервера), загрузите свою базу данных

> ⚠️ **ВАЖНОЕ УТОЧНЕНИЕ**
> Чтобы избежать проблем с загрузкой файлов контейнером нужно загрузить базу данных сразу в папку сервера, хотя бы пустую.

## Шаг 3: HTTPS подлючение

#### 🪟 Инструкция по настройке Cloudpub на Windows

Вы можете использовать любой удобный вариант запуска туннеля на локальной машине: через программу с интерфейсом или через командную строку.

##### Способ А: Через графическое приложение (GUI) (Рекомендуется)
1. Скачайте официальное приложение с графическим интерфейсом для Windows со страницы личного кабинета Cloudpub.
2. Запустите скачанный файл `cloudpub.exe`.
3. Авторизуйтесь в программе, введя логин и пароль от вашего аккаунта.
4. Нажмите на кнопку **«Новая публикация»** (или «Добавить»).
5. Выберите тип ресурса: **HTTP**.
6. Укажите порт локального сервера: **800** (или измените на свой порт, если настраивали другой).
7. Нажмите на кнопку **«Опубликовать»**.
8. Программа отобразит ваш готовый URL-адрес туннеля. Скопируйте его. Не закрывайте приложение во время игры!

##### Способ Б: Через Командную строку (Консоль CLI)
1. Скачайте консольную версию утилиты для Windows и распакуйте архив в удобную папку (например, `C:\cloudpub\`).
2. Откройте **Командную строку** Windows (`Win + R` -> введите `cmd` -> Нажмите `Enter`).
3. Перейдите в папку с утилитой с помощью команды:
   ```cmd
   cd C:\cloudpub
   ```
4. Авторизуйтесь в системе:
   ```cmd
   clo login
   ```
5. Зарегистрируйте постоянный туннель, привязав его к порту локального сервера:
   ```cmd
   clo register http 800
   ```
6. Консоль выведет адрес вашего туннеля. Оставьте это окно открытым на все время работы сервера.

> ℹ️ Имя вашего туннеля можно узнать прямо в интерфейсе приложений при их создании, либо в личном кабинете на сайте в разделе публикации.

## Шаг 4: Запуск сервера

После того как вы настроили все свои файлы, домен и т.п. откройте `server.json` в папке settings

> Настройте следующие параметры:
> ```json
> "port": 800,
> "ping_port": 800,
> "ssl_port": 443,
> "ssl_ping_port": 443,
> "ip": "127.0.0.1",
> "ssl_host": "ваш-туннель"
> ```

> ⚠️ **ВАЖНО**
> Чтобы контейнер каждый раз не грузил карту по новой (что занимает от 10 до 20 минут), мы перенесём её в нашу ОС вручную
> Откройте терминал WSL2 командой `wsl -d Debian` в powershell
> Выполните следующие команды
> ```bash
mkdir -p ~/dynast/maps
cp -r /mnt/c/Users/chlvk/Desktop/NoahNode2/files/maps/* ~/dynast/maps/```

### Команда запуска сервера 
```bash
docker run --rm --name dynast -e DYNASTIO_HETWORK_MODE=host -e DYNASTIO_MAP=standard -e DYNASTIO_LABEL="BymiNoob" -e DYNASTIO_UID=google:112901020510349532837 -e DYNASTIO_PIN_CODE=632-679-288 -p 800:800 -v C:\Users\chlvk\Desktop\files\settings:/usr/local/bin/settings -v C:\Users\chlvk\Desktop\files\data.sqlite:/usr/local/bin/data.sqlite -v C:\Users\chlvk\Desktop\files\logs:/var/log -v \\wsl$\Debian\home\root1\dynast\maps:/usr/local/bin/maps -d n0uk/dynast.io-server-private:release
```  

#### Описание команды: 
* `-e DYNASTIO_MAP` — название карты
* `-e DYNASTIO_LABEL="BymiNoob"` — название вашего сервера
* `-p 800:800` — открытие портов сервера (поменяйте если используете другие)
* `-v /root/dyn_private/data.sqlite:/usr/local/bin/` — путь к дб, настройкам и логам, поменяйте первую часть настройки на путь к вашим файлам вплоть до `C:\Users\chlvk\Desktop\files\` (к примеру, если файлы привата находятся в папке dyn_private на диске C, путь будет C:/dyn_private)
* `-v \\wsl$\Debian\home\root1\dynast\maps:/usr/local/bin/maps` - полный путь к картам
* `--name dynast` — название docker контейнера

### Управление контейнером

* **Команда остановки контейнера:** 
  ```bash
  docker stop dynast
  ```
* **Команда перезагрузки контейнера:**
  ```bash
  docker restart dynast
  ```
  *(Где `dynast` - название вашего контейнера)*

* **Проверка запущенных контейнеров:**
  ```bash
  docker ps
  ```
